# ClassNet 开发文档

## 1. 项目概述

ClassNet 是一个专为班级内部使用设计的网络系统，提供了丰富的功能，包括视频播放、音乐播放、日历、天气、AI聊天、聊天、课程表等应用。本开发文档旨在为开发人员提供详细的技术指导，帮助他们理解项目架构、API设计和开发规范。

## 2. 技术架构

### 2.1 后端架构

- **框架**：FastAPI
- **数据库**：SQLite
- **认证**：JWT
- **缓存**：Redis
- **架构风格**：分层架构

### 2.2 前端架构

- **技术栈**：HTML5, CSS3, JavaScript (ES5/ES6), Bulma组件库（优先使用）
- **架构风格**：模块化设计

### 2.3 前端开发提示
- **优先使用Bulma**：在所有前端开发中，必须优先使用Bulma组件库，只有在Bulma无法满足需求时才进行自定义开发
- **AI开发提示**：当使用AI辅助开发时，直接要求使用Bulma组件库实现功能，无需额外说明
- **组件丰富**：Bulma提供了大量现成的组件，如按钮、卡片、表单、导航等，避免重复开发

## 5. 模块说明

### 5.1 认证模块

- **实现文件**：`backend/app/api/auth.py`, `backend/app/services/auth_service.py`
- **功能**：处理用户注册、登录、密码修改等认证相关功能
- **核心类/函数**：
  - `AuthService.register()`: 用户注册
  - `AuthService.login()`: 用户登录
  - `AuthService.change_password()`: 修改密码
  - `AuthService.verify_token()`: 验证JWT令牌

### 5.2 媒体模块

- **实现文件**：`backend/app/api/media.py`, `backend/app/services/media_service.py`
- **功能**：处理媒体文件管理、文件信息获取等功能
- **核心类/函数**：
  - `MediaService.get_videos()`: 获取视频列表
  - `MediaService.get_audios()`: 获取音频列表
  - `MediaService.get_file_info()`: 获取文件信息
  - `MediaService.search_files()`: 搜索文件

### 5.3 聊天模块

- **实现文件**：`backend/app/api/chat.py`, `backend/app/services/chat_service.py`
- **功能**：处理聊天消息管理、WebSocket连接等功能
- **核心类/函数**：
  - `ChatService.get_messages()`: 获取聊天消息
  - `ChatService.send_message()`: 发送消息
  - `ChatService.handle_websocket()`: 处理WebSocket连接

### 5.4 天气模块

- **实现文件**：`backend/app/api/weather.py`, `backend/app/services/weather_service.py`
- **功能**：处理天气信息获取功能
- **核心类/函数**：
  - `WeatherService.get_weather()`: 获取天气信息
  - `WeatherService.fetch_weather_data()`: 从API获取天气数据

### 5.5 系统模块

- **实现文件**：`backend/app/api/system.py`, `backend/app/services/system_service.py`
- **功能**：处理系统状态监控、资源管理等功能
- **核心类/函数**：
  - `SystemService.get_system_status()`: 获取系统状态
  - `SystemService.get_resource_usage()`: 获取资源使用情况

### 5.6 资源服务器

- **实现文件**：`backend/resource_server/main.py`
- **功能**：提供高性能的媒体文件传输服务
- **核心功能**：

  - 支持Gzip压缩
  - 支持中文文件名
  - 支持带宽限制

1. **安装Python 3.9+**
2. **创建虚拟环境**：
   ```bash
   python -m venv venv
   ```

3. **激活虚拟环境**：
   - Windows: `venv\Scripts\activate`


5. **配置环境变量**：复制 `.env.example` 为 `.env` 并填写相应配置

```python
from pydantic_settings import BaseSettings
from typing import Optional


class Settings(BaseSettings):
    """应用配置"""
    # 服务器配置
    server_host: str = "0.0.0.0"
    server_port: int = 8000
    
    # 数据库配置
    database_url: str = "sqlite:///./data/database/classnet.db"
    
    # 认证配置
    secret_key: str
    algorithm: str = "HS256"
    access_token_expire_minutes: int = 1440
    
    # Redis配置
    redis_url: str = "redis://localhost:6379"
    
    # 媒体配置
    media_dir: str = "./data/media"
    
    # 资源服务器配置
    resource_server_port: int = 8001
    
    class Config:
        env_file = ".env"


settings = Settings()
```

## 10. 安全规范

### 10.1 密码安全

- 使用bcrypt对密码进行哈希存储
- 密码长度至少8位
- 密码应包含字母、数字和特殊字符

### 10.2 认证安全

- 使用JWT进行无状态认证
- 令牌设置合理的过期时间
- 实现令牌刷新机制

### 10.3 输入验证

- 对所有用户输入进行验证
- 使用Pydantic进行数据验证
- 防止SQL注入、XSS等攻击

### 10.4 CORS配置

- 正确配置CORS头，允许跨域请求
- 在生产环境中限制允许的源

### 10.5 日志安全

- 日志中不包含敏感信息
- 实现日志轮转，防止日志文件过大
