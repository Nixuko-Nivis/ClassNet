# 开发环境搭建指南

## 1. 安装VS Code

1. 访问 [VS Code官网](https://code.visualstudio.com/) 下载最新版本的Visual Studio Code
2. 运行安装程序，按照向导完成安装

## 2. 安装必要插件

打开VS Code后，点击左侧的扩展图标（或按 `Ctrl+Shift+X`），搜索并安装以下插件：

### 2.1 核心插件

- **Live Server**：本地开发服务器，用于预览前端页面
- **Prettier - Code formatter**：代码格式化工具，保持代码风格一致
- **ESLint**：代码质量检查工具，帮助发现和修复代码问题
- **HTML CSS Support**：增强HTML和CSS的编辑体验

### 2.2 推荐插件

- **Auto Rename Tag**：自动重命名HTML标签
- **Bracket Pair Colorizer**：为括号添加颜色，提高代码可读性
- **Code Spell Checker**：拼写检查工具
- **GitLens**：增强Git功能

## 3. 配置VS Code

### 3.1 配置Prettier

1. 打开设置（`File > Preferences > Settings` 或 `Ctrl+,`）
2. 搜索 "Prettier"
3. 勾选 "Format On Save"，使Prettier在保存文件时自动格式化代码

### 3.2 配置ESLint

1. 打开设置
2. 搜索 "ESLint"
3. 勾选 "ESLint: Enable"
4. 勾选 "ESLint: Auto Fix On Save"

## 4. 验证开发环境

1. 打开项目目录 `c:\Users\Administrator\Documents\ClassNet`
2. 确保VS Code能够正确识别项目结构
3. 安装插件后，重启VS Code以确保所有插件生效

## 5. 启动开发服务器

1. 在VS Code中打开前端目录 `frontend`
2. 右键点击 `index.html` 文件（如果存在），选择 "Open with Live Server"
3. 或在编辑器中点击右下角的 "Go Live" 按钮
4. Live Server会启动一个本地服务器，并在浏览器中打开页面

## 6. 开发流程

1. 在VS Code中编辑代码
2. 保存文件时，Prettier会自动格式化代码
3. ESLint会检查代码质量并提示问题
4. Live Server会自动刷新浏览器，显示最新的更改

## 7. 故障排除

### 7.1 Live Server无法启动
- 检查端口是否被占用
- 尝试在不同的端口启动Live Server

### 7.2 Prettier不工作
- 确保已安装Prettier插件
- 检查文件是否有正确的扩展名
- 检查设置是否正确

### 7.3 ESLint报错
- 查看错误信息，根据提示修复代码
- 如需忽略某些规则，可在代码中添加注释

## 8. 总结

搭建好开发环境后，您可以开始开发前端代码。VS Code及其插件将帮助您提高开发效率，确保代码质量。
