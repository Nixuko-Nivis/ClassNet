# ClassNet 项目重构计划文档

## 1. 项目重构概述

本计划旨在提供一份全面的项目重构方案，该方案将创建一个与现有项目完全不同的架构实现，同时保留所有原有功能。本次重构允许使用全新的技术栈和框架（前端技术栈保持不变），并采用新的目录结构和技术实现方式。因为服务器都在同一台设备上，所以不使用nginx和docker。

## 2. 重构目标

- 可以创建与现有项目不同的架构实现
- 保留并完整实现所有原有功能
- 保持前端技术栈不变
- 允许使用全新的后端技术栈和框架
- 设计并实现全新的项目目录结构
- 提升代码质量、可维护性和可扩展性
- 优化性能和用户体验

## 3. 技术栈规划

### 3.1 前端技术栈（保持不变）
- HTML5, CSS3, JavaScript (ES5/ES6)
- Bulma组件库
- 保持现有前端框架、库和工具链

### 3.2 后端技术栈（全新）
- **编程语言**：Python 3.9+
- **框架**：FastAPI
- **数据库**：SQLite
- **API设计**：RESTful API
- **认证授权**：JWT (JSON Web Tokens)
- **缓存策略**：Redis
- **消息队列**：None（当前项目规模不需要）

## 4. 项目架构设计

### 4.1 整体架构

新的系统架构采用分层设计，包括以下层次：

1. **前端层**：保持现有前端代码结构，通过HTTP请求与后端API通信
2. **API层**：FastAPI实现的RESTful API，处理HTTP请求和响应
3. **服务层**：业务逻辑层，处理核心业务逻辑
4. **数据层**：数据访问层，处理数据库操作
5. **资源层**：独立的资源服务器，优化媒体文件传输

**数据流**：
- 前端请求 → API层 → 服务层 → 数据层 → 服务层 → API层 → 前端响应
- 媒体资源请求 → 资源服务器 → 前端

## 6. 开发流程与规范

### 6.1 开发环境配置

1. **安装Python 3.9+**
2. **创建虚拟环境**：
   ```bash
   python -m venv venv
   ```
3. **激活虚拟环境**：
   - Windows: `venv\Scripts\activate`
   - Linux/Mac: `source venv/bin/activate`
4. **安装依赖**：
   ```bash
   pip install -r backend/requirements.txt
   ```
5. **配置环境变量**：复制 `.env.example` 为 `.env` 并填写相应配置

### 6.2 编码规范

- **Python**：遵循PEP 8规范
- **命名约定**：
  - 类名：大驼峰命名法
  - 函数和变量名：小写字母加下划线
  - 常量：全大写字母加下划线
- **代码注释**：为复杂逻辑添加注释，使用文档字符串
- **错误处理**：使用try-except捕获异常，返回适当的错误响应

### 6.3 版本控制策略

- **分支管理**：
  - main：主分支，用于发布
  - develop：开发分支
  - feature/xxx：功能分支
  - fix/xxx：修复分支
- **提交规范**：
  - 提交信息格式：`[类型] 描述`
  - 类型：feat（新功能）、fix（修复）、docs（文档）、style（格式）、refactor（重构）、test（测试）、chore（构建/依赖）

### 6.4 代码审查流程

1. **提交前检查**：运行代码格式化工具和静态检查工具
2. **创建Pull Request**：从功能分支向develop分支创建PR
3. **代码审查**：至少有1名团队成员审查代码
4. **合并**：审查通过后合并到develop分支
5. **测试**：合并后运行测试

## 7. 测试计划

### 7.1 单元测试策略

- **框架**：pytest
- **覆盖范围**：
  - 服务层核心逻辑
  - 工具函数
  - 数据模型
- **运行方式**：
  ```bash
  pytest backend/tests/
  ```

### 7.2 集成测试策略

- **工具**：pytest + FastAPI TestClient
- **测试内容**：
  - API端点测试
  - 数据库集成测试
  - 认证流程测试

### 7.3 性能测试计划

- **指标**：
  - API响应时间
  - 资源服务器传输速度
  - 并发处理能力
- **工具**：Locust
- **测试场景**：
  - 并发用户访问
  - 大文件传输
  - 聊天消息处理

### 7.4 用户验收测试

- **测试流程**：
  1. 功能测试：验证所有功能正常工作
  2. 兼容性测试：验证在不同浏览器和WebView环境下的兼容性
  3. 性能测试：验证系统在正常负载下的性能
- **测试标准**：
  - 所有功能正常工作
  - 响应时间合理
  - 无明显bug

## 8. 部署与发布策略

### 8.1 部署架构

- **服务器**：同一台设备上运行多个服务
- **端口分配**：
  - 主服务器：8000
  - 资源服务器：8001
- **依赖管理**：使用pip和requirements.txt

### 8.2 CI/CD流程

- **构建**：运行依赖安装和测试
- **测试**：运行单元测试和集成测试
- **部署**：更新服务器代码并重启服务

### 8.3 发布策略

1. **准备**：确保所有测试通过
2. **备份**：备份当前运行的代码和数据库
3. **部署**：更新代码并重启服务
4. **验证**：验证服务正常运行
5. **回滚**：如果出现问题，恢复备份
